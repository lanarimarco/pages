Piano di Migrazione RDS

Da VSS a GitHub con Maven

Questo documento delinea i passaggi, le tempistiche e le azioni necessarie per la transizione del progetto RDS al nuovo ambiente di sviluppo, in preparazione per il progetto Menarini ODP 3.0.

Obiettivi della Migrazione

  • Modernizzare il workflow: Abbandonare VSS in favore di Git e GitHub per una gestione del codice più efficiente e collaborativa.
  • Semplificare il setup: "Mavenizzare" il progetto RDS per ridurre drasticamente la complessità del primo setup per gli sviluppatori.
  • Migliorare la Toolchain: Introdurre strumenti moderni come un server Tomcat embedded per test e un sistema di patching più robusto.
  • Allineamento Strategico: Preparare la codebase e il team per le sfide del nuovo progetto Menarini ODP 3.0.

Ambiente di Sviluppo

La codebase è sincronizzata ogni notte da VSS al seguente repository GitHub. Questo repository diventerà la nostra unica fonte di verità.

Repository Principale:

https://github.com/lanarimarco/rdsv6

Branch di Lavoro Principale (in corso):

feature/mavenizedrdsv6

Branch Sperimentali

Per consentire a tutti di prendere confidenza con il nuovo ambiente, sono state create delle branch personali. Usatele per i vostri esperimenti e per seguire i video tutorial.

  1. Clonare il repository lanarimarco/rdsv6.
  2. Eseguire il checkout della propria branch personale (es. test/andrea).
  3. Attendere e seguire i video tutorial non appena saranno disponibili.
