‚úÖ Advantages

  • Complete conversion in one session
  • Better context retention
  • Faster overall execution
  • Fewer manual steps

‚ö†Ô∏è Considerations

  • Requires large context window

# ‚Üí Execute in LLM, review output

./run-developer.sh CPY001 PGM001
cat work-in-progress/CPY001/30-developer/PROMPT.md | pbcopy
# ‚Üí Execute in LLM, review output

‚úÖ Advantages

  • Better control over each phase
  • Can review before proceeding
  • Works with smaller context windows
  • More cost-effective

‚ö†Ô∏è Considerations

  • More manual steps required

Troubleshooting Guide

Problem Solution
LLM runs out of context Use phase-by-phase mode instead of master prompt. Break large programs into smaller chunks.
LLM doesn't understand RPGLE Enhance analysis phase with more context. Add RPGLE patterns to common-patterns/
Generated code has compilation errors Run mvn compile to identify issues. Review and fix syntax errors. Check imports and dependencies.
Tests fail Review test logic against RPGLE behavior. Update test data. Check edge cases. Run mvn test -X for details.
Missing business logic in conversion Improve analysis phase documentation. Add detailed notes about complex logic. Review architect phase plan.
Type conversion issues Check type-mappings.md. Verify packed decimal ‚Üí BigDecimal conversions. Review numeric precision.
Parameter handling incorrect Verify program entry point implementation. Check parameter reading/writing logic. Review Kokos SDK docs.
