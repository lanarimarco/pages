L'inserimento dei test nel JAR di deploy è fortemente sconsigliato per preservare la pulizia dell'ambiente di runtime.

Delegare l'esecuzione a GitHub garantisce che le simulazioni non siano influenzate da variabili d'ambiente locali.

Strategia di Test (Doppio Controllo)

Abbiamo adottato due tipologie di test per massimizzare la stabilità:

1. Test da Simulazione: Creato fedelmente dal file .sch per la parità con il legacy.
2. Test Business Logic: Unit test estesi per verificare casi limite e logiche atomiche.

Questo approccio permette di individuare istantaneamente eventuali casi contraddittori tra specifica e implementazione.
